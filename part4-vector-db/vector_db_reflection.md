## Vector DB Use Case

A traditional keyword-based database would not be sufficient for searching through large legal contracts using natural language queries. Keyword search relies on exact word matches, which can fail when the same concept is expressed using different wording. For example, a clause related to “termination” might not explicitly use that exact word but instead use phrases like “ending the agreement” or “contract expiration.” In such cases, keyword search would miss relevant results, reducing accuracy and efficiency.

A vector database, on the other hand, uses embeddings to capture the semantic meaning of text. This allows the system to understand the intent behind a query rather than just matching words. When a lawyer asks a question in plain English, the system converts both the query and contract text into vectors and retrieves the most semantically similar sections. This makes the search more intelligent and context-aware.

Additionally, vector databases scale well for large documents and enable faster similarity searches. They are particularly useful when dealing with unstructured data like legal text.

Therefore, a vector database would play a crucial role in improving search accuracy, speed, and usability in such a system. It enables lawyers to interact naturally with complex documents without needing precise keyword knowledge.
